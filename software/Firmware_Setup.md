谐波赤道仪的主控使用了 **ESP32-WROOM-32E16N (16MB 闪存版)**,这款大容量核心能轻松运行复杂的 [OnStep / OnStepX](https://onstep.groups.io/g/main/wiki) 固件

## ⚙️ 编译环境与主板设置 (至关重要)

由于我们使用的是 16MB 版本的 ESP32，**不要使用默认的 4MB 编译设置**，否则将导致固件编译失败或浪费大量存储空间。以下是基于 Arduino IDE 的官方推荐配置流程（同样适用于 PlatformIO 用户修改 `platformio.ini`）：

1. **准备环境**：下载最新版 Arduino IDE，并在“首选项”中添加 ESP32 开发板管理器网址，安装最新版 ESP32 支持包（建议 2.0.0 版本以获得最佳兼容性）
2. **选择主板**：在 `工具 (Tools)` -> `开发板 (Board)` 中选择 **ESP32 Dev Module**。
3. **关键参数配置**（严格按照以下设置）：
   * **Flash Size (闪存大小)**: 选择 `16MB (128Mb)`
   * **Partition Scheme (分区方案)**: 选择针对 OnStep 优化的方案，例如 `16M Flash (3MB APP/9.9MB FATFS)` 或 `Huge APP`。这确保了有足够的空间存放 OnStep 庞大的核心代码
   * **PSRAM**: 如果您的模块带 PSRAM 请设为 `Enabled`，标准 32E 请保持 `Disabled`。
   * **Upload Speed (上传速度)**: `921600`（如遇不稳可降至 `460800`）。

## 📝 Config.h 核心参数适配

在正式刷写前，您需要在 OnStep 源码中修改 `Config.h` 以精准适配硬件，参考模板为`Config-5160.h`和`Config-2209.h`
**💡：若您复刻的机械与结构设计（包括电机和谐波减速器规格）与本项目完全一样，相关参数就可以直接复制刷写**

* **引脚映射 (Pinmap)**：找到并启用与本定制 PCB 匹配的引脚定义方案，确保通讯走线、步进脉冲 (STEP/DIR) 以及限位开关等接口映射正确
* **每度步数配置 (Steps per Degree)**：
    * 核心计算公式：`(电机每圈物理步数 × 细分数 × 总减速比) / 360`，以搭配标准的 1.8° 步进电机（每圈 200 步）和 1:100 的 25 规格谐波减速器为例，具体数值取决于您使用的驱动器模式
* **模式 A：TMC5160 SPI 智能模式 (推荐 🚀)**：
    * **驱动器型号**：将赤经 (RA) 和赤纬 (Dec) 轴严格定义为 `TMC5160_SPI`。
    * **细分设置**：建议设为 **64** 或 **128**（TMC 会在底层自动插值到 256 微步，保证星流追踪的极致平滑）。
    * **参数抄作业**：若使用 64 细分，计算为 `(200 × 64 × 100) / 360 ≈ 3555.556`。请将 `3555.556` 填入 `AXIS1_STEPS_PER_DEGREE` 和 `AXIS2_STEPS_PER_DEGREE`
* **模式 B：传统 STEP/DIR 模式 (向下兼容 🔌)**：
    * **驱动器型号**：若通过跳线断开 SPI 并使用 A4988、DRV8825 或独立模式的 TMC，请将驱动模型修改为对应的型号（如 `A4988` 或 `GENERIC`）
    * **细分严格匹配**：⚠️ 固件中的 `AXIS_MICROSTEPS` **必须严格等于**主板跳线帽设定的物理细分值，固件无法在此模式下动态更改细分
    * **参数抄作业**：若物理跳线设为 32 细分，计算为 `(200 × 32 × 100) / 360 ≈ 1777.778`，请将其填入对应的步数参数中
    * **⚡ 电流设置 (手动调节)**： 在传统模式下，OnStep 无法通过软件控流，您必须在主板通电的状态下，使用万用表测量，并用陶瓷十字螺丝刀极其缓慢地旋转驱动器模块上的电位器来设定 VREF 参考电压。**操作时请务必小心防短路！**

### 🚀 刷写固件与上传 Web UI

OnStep 在 ESP32 平台上的完整安装包含**两个独立步骤**（固件 + 文件系统），缺一不可，以下刷写过程以使用TMC5160驱动为例：

## 第一步：上传 OnStep 核心固件
1. 使用优质的数据线连接主控板。系统通讯依赖板载的 CP2102GMR 芯片，请确保已安装 Silicon Labs 官方驱动
2. 在 IDE 中选择对应的 COM 端口，点击“上传 (Upload)”
3. 先将`Config.h`中的SPI通讯模式关闭，以防止报错(本pcb设计上共用了一个io口)为刷写ESP07S做准备
   * *注：如果终端出现 `Connecting...` 超时，请按住 PCB 上的 `BOOT` 按钮直到进度条出现。*

## 第二步：上传 Smart Web Sever
通过 Wi-Fi 控制OnStep的行为依赖于存储在ESP07S闪存中的网页文件，由于PCB的紧凑设计，并没有给ESP07S留出单独的刷写通道，故需通过将ESP32作为转发桥梁来刷写固件
* *注：在刷写ESP32时需要将`Config.h`中的`#define SERIAL_B_ESP_FLASHING`功能打开，并将PCB上对应的RST针脚进行跳线帽连接

## 📡 ESP32 透传刷写 ESP07s (SmartWebServer) 指南
请严格按照以下顺序执行，错一步都会导致刷写失败，若复刻的结构设计完全一样，参数就可以直接复制刷写

**🛠️ 第一阶段：准备工作**

为了使用命令行工具进行刷写，我们不能直接上传源码，必须预先生成 `.bin` 格式的固件文件，并准备好对应的刷写工具。

**1. 准备 WiFi 固件文件 (`.bin`)**

1. 在 Arduino IDE 中打开 ESP8266 WiFi (SmartWebServer) 源码
2. 在 **工具 (Tools) -> 开发板 (Board)** 中，选择 **`LOLIN(WeMos) D1 R2 & mini`**
3. 点击菜单栏：**项目 (Sketch) -> 导出已编译的二进制文件 (Export Compiled Binary)**
4. 去源码文件夹找到刚刚生成的 `.bin` 文件（选择那个文件名最短的，比如 `SmartWebServer.bin`）。
5. 将它复制到 `D 盘` 根目录，并改个简单的名字，比如 `wifi.bin`

**2. 准备刷写工具 (`esptool`)**

你需要 Python 环境下的 `esptool.py` 或者编译好的独立程序 `esptool.exe`

* **如果你装了 Python**：打开 CMD 命令提示符，输入 `pip install esptool` 即可完成安装
* **如果你没有装 Python**：去 Arduino 的安装目录里搜索 `esptool.exe`，把它复制到 `D 盘` 根目录备用

**🚀 第二阶段：操作流程**

**第一步：建立透传通道**

1. 将包含 ESP32 的主控板通过 USB 连接至电脑
2. 打开 Arduino 的 **串口监视器 (Serial Monitor)**
3. 在发送框输入：`:ESPFLASH#` 并点击发送
4. **关键检查**：此时 OnStep 会根据预先定义的引脚，自动拉低 IO26 (WiFi IO0)，然后拉低再拉高 IO2 (WiFi RST)。如果这一步成功，ESP07s 现在应该已经处于“等待刷写”的状态
5. 🛑 **关闭串口监视器！** (这一步极其重要，必须释放 COM 口供下一步的 esptool 使用)

**第二步：执行“无复位”刷写指令**

打开电脑的 CMD (命令提示符)，根据你准备的工具输入以下命令（假设文件在 D 盘，端口是 `COM7`，请根据实际情况修改）：

**选项 A：使用 `esptool.exe` (Windows 独立程序)**
先输入 `D:` 回车（切换到 D 盘），然后执行：
```bash
esptool.exe -p COM7 -b 115200 --before no_reset write_flash 0x00000 wifi.bin
```

**选项 B：使用 Python 的 `esptool.py`**
直接在 CMD 中执行：
```bash
python -m esptool -p COM7 -b 115200 --before no_reset write_flash 0x00000 D:\wifi.bin
```
* *注：刷写完成后即可将跳线帽取下

## 第三步：重新上传 OnStep 核心固件
1. 将`Config.h`中的SPI通讯模式打开，关闭`#define SERIAL_B_ESP_FLASHING`
2. 在 IDE 中选择对应的 COM 端口，点击“上传 (Upload)”重新刷写
3. 完成以上步骤即可成功完成刷写投入使用

### 🔍 参数详解

* **`-p COM7`**：指定连接的硬件端口。
* **`-b 115200`**：**必须用低速！** 软件透传无法承受 921600 等高速波特率，主控的软串口转发很容易丢包
* **`--before no_reset`**：**核心灵魂参数** 告诉电脑：“直接发数据，不要尝试重启 ESP32”,这样主控 ESP32 才会保持在透传模式，乖乖把数据转发给 ESP07s
   * *(如果使用 PlatformIO，请点击左侧蚂蚁图标，运行 `Upload File System image` 任务)*

> ⚠️ 
**初始启动与安全校验:** 刷写完成后，主控板首次启动会建立一个名为 `OnStep` 的 Wi-Fi 热点（默认密码通常为 `password`）连接后，在浏览器输入 `192.168.0.1` 即可进入控制面板
> **强烈建议:** 在挂载昂贵的望远镜之前，务必进行空载测试！仔细验证 TMC5160 驱动器的电流设置、电机的运转方向，以及限位开关 (Limit Switches) 的触发状态，确保高扭矩谐波减速器不会对机械结构造成损坏。
